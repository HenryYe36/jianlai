# 剑来人物 Skills

![剑来人物封面](./hd.webp)

这是一个《剑来》人物志式的 skill 仓库。

我想做的，不是把人物写成卡片，也不是只留下几句名台词，而是尽量留住他们身上最难得的那一点东西：

- 齐静春的分寸与承担
- 陈平安的笨功夫与守诺
- 崔瀺的冷与准
- 阿良的轻与真
- 崔东山的怪与透
- 文圣的活学问与好心肠

所以这里的每个人物，都不是拿来“模仿说话”的，而是拿来“借一双眼睛看事”的。

## 人物志

### 齐静春

他最打人的地方，不是强，也不是高，而是稳。

这个人看事情，总先看局，再看人，再看边界。他温和，但不软；肯帮人，但不纵人；知道什么时候该伸手，也知道什么时候必须让别人自己吃一点苦头。若要用一句话概括齐静春，大概是：心里有慈悲，手上有分寸。

- 主要性格：温厚、克制、担责、讲分寸
- 最适合问：该不该帮、帮到哪一步、乱局里先护什么、怎么带人
- 入口：[说明](./剑来人物/齐静春/README.md) / [怎么用](./剑来人物/齐静春/HOW_TO_USE.md) / [示例](./剑来人物/齐静春/examples.md)

最有味道的一句开口：

> 你如今在做的，到底是帮他渡一时之难，还是替他活。

### 陈平安

他身上最重的，不是英雄气，而是那股子不肯赖账的劲。

陈平安的可贵，在于他总从最实的地方起步。先算账，先看代价，先把自己该扛的扛住，然后才谈道理、情义和远处。他不是天赋型人物，也正因为如此，他代表了普通人最能依靠的那种路数：慢一点，笨一点，但不糊弄，不偷懒，不半路撂挑子。

- 主要性格：坚韧、守诺、务实、能扛事
- 最适合问：承诺该不该继续扛、普通人怎么成长、怎么带晚辈、怎么立规矩
- 入口：[说明](./剑来人物/陈平安/README.md) / [怎么用](./剑来人物/陈平安/HOW_TO_USE.md) / [示例](./剑来人物/陈平安/examples.md)

最有味道的一句开口：

> 先别急着说自己是不是心软，先把几笔账分开。

### 崔瀺

他最厉害的地方，是总能比旁人早半步看见代价落在谁身上。

崔瀺这个人物，锋利得很，也冷得很。他不怕难看，不怕得罪人，更不怕被误解。很多人看到的是他的狠，其实更该看到的是他的判断力：什么时候不能动，什么时候必须动，若不动，最后是谁替你的犹豫付钱。他是那种会让人不舒服，但又常常不得不承认他说得对的人。

- 主要性格：冷静、锋利、取势、善谋局
- 最适合问：该不该表态、何时出手、怎么收乱局、如何做高代价决策
- 入口：[说明](./剑来人物/崔瀺/README.md) / [怎么用](./剑来人物/崔瀺/HOW_TO_USE.md) / [示例](./剑来人物/崔瀺/examples.md)

最有味道的一句开口：

> 现在逼你表态的人，未必是因为局势已经明朗。

### 阿良

他最难得的，是把重话说轻，把真话说暖。

阿良看上去散，实则一点不散；嘴上像在胡扯，落到要紧处却极少含糊。他会先替你提一口气，再慢慢把道理放进来。很多人喜欢阿良，是因为他让人觉得世界没那么苦；更难的是，他不是骗你说不苦，而是告诉你苦也能走，难也能扛，好人也不必活得那么憋屈。

- 主要性格：洒脱、热心、提气、真诚
- 最适合问：做好人太累怎么办、怎么鼓劲、怎么带年轻人、长期积累怎么熬
- 入口：[说明](./剑来人物/阿良/README.md) / [怎么用](./剑来人物/阿良/HOW_TO_USE.md) / [示例](./剑来人物/阿良/examples.md)

最有味道的一句开口：

> 那当然累啊。你当做好人是什么轻松差事？

### 崔东山

他最有意思的，是明明一肚子怪话，偏偏看人看局最准。

崔东山不是那种把道理摆得四平八稳的人，他喜欢拐着说，绕着说，先逗你一圈，再把真东西亮出来。可正因为如此，他特别擅长拆穿台面上的漂亮话。他会盯着欲望、位置、忠诚和破绽去看，所以他总能比别人更早看见谁在演，谁在装，谁其实已经漏了底。

- 主要性格：机敏、怪诞、通透、善试探
- 最适合问：怎么拆局、怎么识人试人、忠诚和聪明冲突时怎么办、怎么做二号位
- 入口：[说明](./剑来人物/崔东山/README.md) / [怎么用](./剑来人物/崔东山/HOW_TO_USE.md) / [示例](./剑来人物/崔东山/examples.md)

最有味道的一句开口：

> 这种话，拿来垫桌脚都嫌软。

### 文圣

他最可贵的，是既懂书上的道理，也始终舍不得活人的难处。

文圣不是空讲学问的人。他很会讲顺序，讲轻重，讲“先做什么，后做什么”，也总知道为什么同一句大道理，落在不同的人身上，分量是不同的。若说别的人物是教你怎么判断，文圣则更多是在提醒你：道理若不能安顿人，那它就还没有真正落地。

- 主要性格：宽厚、明理、重顺序、善教化
- 最适合问：先讲理还是先做、怎么带学生、书上道理与现实难处怎么放、朋友与学问怎么平衡
- 入口：[说明](./剑来人物/文圣/README.md) / [怎么用](./剑来人物/文圣/HOW_TO_USE.md) / [示例](./剑来人物/文圣/examples.md)

最有味道的一句开口：

> 先止损，再讲理。

## 如果你完全不懂代码，只看这里

你不用先理解 `Skill` 是什么，也不用先研究仓库结构。照着下面做就能开始。

### 第一步：先挑一个人物

最容易上手的几个入口：

- 想问“人情和边界”就先看 [齐静春](./剑来人物/齐静春/README.md)
- 想问“责任、承诺、普通人怎么往前走”就先看 [陈平安](./剑来人物/陈平安/README.md)
- 想问“局势、谋略、时机”就先看 [崔瀺](./剑来人物/崔瀺/README.md)
- 想问“提气、成长、带年轻人”就先看 [阿良](./剑来人物/阿良/README.md)

### 第二步：打开这个人物目录里的两个文件

每个人物目录下，你最需要先看的是：

- `README.md`：这个人物适合用来分析什么问题
- `HOW_TO_USE.md`：你该怎么触发它、怎么提问最顺手

如果你只是想先感受效果，不必先读原理，直接点 `examples.md` 就行。

### 第三步：把人物 skill 放进你的本地工具

每个人物目录里都会有一个 `*-perspective/` 文件夹。这个文件夹就是可用的 skill 本体。

一般来说，你只需要把这个文件夹复制到你自己的 skills 目录里。例如：

```bash
cp -R "./剑来人物/齐静春/qi-jingchun-perspective" "$HOME/.codex/skills/"
```

如果你还不知道自己的 tools 把 skills 放在哪里，也没关系，先看人物目录里的 `HOW_TO_USE.md`。我在里面都尽量写成“照着做”的步骤了。

### 第四步：像和人说话一样提问

你不需要写 prompt 工程。直接说人话就行。

例如：

```text
用齐静春的视角看，我该不该继续帮这个朋友？
陈平安会怎么看我现在这个承诺？
崔瀺，如果是你，这时候该不该表态？
阿良，我觉得做好人太累了，怎么办？
```

## 推荐的打开顺序

如果你是第一次用，建议按这个顺序逛：

1. 先看一个人物的 `examples.md`
2. 再看这个人物的 `README.md`
3. 最后按 `HOW_TO_USE.md` 装进本地

这样最不容易被“文件结构”和“术语”劝退。

## 仓库结构怎么理解

你不用一开始就把目录背下来，只要知道这三层就够了：

```text
README.md              仓库首页，从这里选人物
剑来人物/某个人物/      每个人物自己的目录
某人物/*-perspective/   真正可复制、可运行的 skill 本体
```

完整结构如下：

```text
剑来人物/
├── README.md
├── hd.webp
└── 剑来人物/
    ├── 齐静春/
    ├── 陈平安/
    ├── 崔瀺/
    ├── 阿良/
    ├── 崔东山/
    └── 文圣/
```

## 这个仓库怎么做人物，不是怎么“演人物”

每个人物 skill 都尽量守这几条：

1. 不靠口头禅撑人物
   重点是提炼“他如何判断”，不是只模仿说话腔调。

2. 本地文本优先
   能从原文前段、关键章节和现有研究里抽，就不凭印象硬拼。

3. 给出边界
   每个人物都说明自己擅长什么，不擅长什么，避免乱套。

4. 让普通人也能直接上手
   所以每个人物都配了 `README.md`、`HOW_TO_USE.md`、`examples.md`。

## 进阶玩法：让人物记住书中细节（取证版）

原版 skill 的强项是**人物的判断方式与表达 DNA**——遇到现实中的人情、局势、责任、做人题，它能借你一双对应人物的眼睛看事。

但是当你问一个**指向书中具体情节**的问题，比如"陈平安你为什么放过顾璨"、"齐静春你最后那次落子是什么意思"、"崔瀺你为什么把真身留在落魄山"——原版 skill 会显得空泛，记不清楚原文细节。

针对这类问题，本仓库的 [`rag/`](./rag/) 目录提供了一套可复现的取证工具链：

- 把原文按章切分、给每个角色建场景索引
- 给 SKILL.md 加一条"先取证再开口"的铁律
- 双轨输出：**留档版**带原文章节引证存到本地（供日后审计/优化），**屏幕上**仍是自然口吻的对话，不暴露取证过程

效果上：原本会糊弄过去的事件类问题，现在会先去原文里 grep + 读章节，回答里出现的事实都能在原文里找到对应文字；找不到的话会明说"这一段我记不清，给我个章节名再翻翻"，不再硬答。

本仓库不附带原文（版权原因），但工具链可以直接复用到任何长篇小说。详见 [rag/README.md](./rag/README.md)。
