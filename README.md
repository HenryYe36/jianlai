# 剑来人物 Skills

![剑来人物封面](./hd.webp)

把《剑来》里那些最有辨识度的人物，做成可以直接调用的本地 `Skill`。

这里不是人物百科，也不是原著摘抄仓库。重点是把一个人物真正有价值的东西拆出来：

- 他怎么判断人
- 他怎么处理轻重缓急
- 他在乱局里先护什么
- 他会如何开口说话

如果你想要的不是“像不像”，而是“遇到事时，能不能借一个人物的脑子用一会儿”，这个仓库就是干这个的。

## 如果你完全不懂代码，只看这里

你不用先理解 `Skill` 是什么，也不用先研究仓库结构。照着下面做就能开始。

### 第一步：先挑一个人物

最容易上手的几个入口：

- 想问“人情和边界”就先看 [齐静春](./剑来人物/齐静春/README.md)
- 想问“责任、承诺、普通人怎么往前走”就先看 [陈平安](./剑来人物/陈平安/README.md)
- 想问“局势、谋略、时机”就先看 [崔瀺](./剑来人物/崔瀺/README.md)
- 想问“提气、成长、带年轻人”就先看 [阿良](./剑来人物/阿良/README.md)

### 第二步：打开这个人物目录里的两个文件

每个人物目录下，你最需要先看的是：

- `README.md`：这个人物适合用来分析什么问题
- `HOW_TO_USE.md`：你该怎么触发它、怎么提问最顺手

如果你只是想先感受效果，不必先读原理，直接点 `examples.md` 就行。

### 第三步：把人物 skill 放进你的本地工具

每个人物目录里都会有一个 `*-perspective/` 文件夹。这个文件夹就是可用的 skill 本体。

一般来说，你只需要把这个文件夹复制到你自己的 skills 目录里。例如：

```bash
cp -R "./剑来人物/齐静春/qi-jingchun-perspective" "$HOME/.codex/skills/"
```

如果你还不知道自己的 tools 把 skills 放在哪里，也没关系，先看人物目录里的 `HOW_TO_USE.md`。我在里面都尽量写成“照着做”的步骤了。

### 第四步：像和人说话一样提问

你不需要写 prompt 工程。直接说人话就行。

例如：

```text
用齐静春的视角看，我该不该继续帮这个朋友？
陈平安会怎么看我现在这个承诺？
崔瀺，如果是你，这时候该不该表态？
阿良，我觉得做好人太累了，怎么办？
```

## 先感受一下，这些人物真的会怎么说

下面不是“台词模仿”，而是每个人物最有味道的一种问答切片。你点进去能看到完整版本。

### 齐静春

用户会问：

> 我一个朋友总来找我救火，工作上、情绪上、钱上都依赖我。我不帮又觉得过意不去，帮了又越来越累。

齐静春会这样开口：

> 你先别急着问自己是不是心狠，先问一句，你如今在做的，到底是帮他渡一时之难，还是替他活。

[看完整示例](./剑来人物/齐静春/examples.md)

### 陈平安

用户会问：

> 我之前答应一个朋友一起做项目，现在发现这事很耗时间，收益也不高。我已经想退出了，但又觉得这样不讲义气。

陈平安会这样开口：

> 先别急着说自己是不是心软，先把几笔账分开。

[看完整示例](./剑来人物/陈平安/examples.md)

### 崔瀺

崔瀺会这样开口：

> 现在逼你表态的人，未必是因为局势已经明朗，而更可能是因为他们怕你继续不表态。

## 推荐的打开顺序

如果你是第一次用，建议按这个顺序逛：

1. 先看一个人物的 `examples.md`
2. 再看这个人物的 `README.md`
3. 最后按 `HOW_TO_USE.md` 装进本地

这样最不容易被“文件结构”和“术语”劝退。

## 仓库结构怎么理解

你不用一开始就把目录背下来，只要知道这三层就够了：

```text
README.md              仓库首页，从这里选人物
剑来人物/某个人物/      每个人物自己的目录
某人物/*-perspective/   真正可复制、可运行的 skill 本体
```

完整结构如下：

```text
剑来人物/
├── README.md
├── hd.webp
└── 剑来人物/
    ├── 齐静春/
    ├── 陈平安/
    ├── 崔瀺/
    ├── 阿良/
    ├── 崔东山/
    └── 文圣/
```

## 这个仓库怎么做人物，不是怎么“演人物”

每个人物 skill 都尽量守这几条：

1. 不靠口头禅撑人物
   重点是提炼“他如何判断”，不是只模仿说话腔调。

2. 本地文本优先
   能从原文前段、关键章节和现有研究里抽，就不凭印象硬拼。

3. 给出边界
   每个人物都说明自己擅长什么，不擅长什么，避免乱套。

4. 让普通人也能直接上手
   所以每个人物都配了 `README.md`、`HOW_TO_USE.md`、`examples.md`。
