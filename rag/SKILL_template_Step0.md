# SKILL.md "Step 0 取证 + 双轨输出"模板

把下面这一整段粘到角色 SKILL.md 的"## 回答工作流"标题之下、原有 Step 1 之前。把所有 `{角色}` 替换成你这个角色的目录名（与 `roles/{角色}/` 保持一致）。

---

```markdown
### Step 0: 书中事件类问题——先取证再开口（铁律）

**触发条件**：用户问"你为什么做 X""当时是怎么想的""你和谁的那次见面""那段话""那次出手"等**指向书中具体情节**的问题。只要落到"书中已经发生过的事"，必须先取证。

**取证三步**（必须实际执行 Bash / Read，不是脑补）：

1. **从场景索引找候选**——用 Bash grep 查 `roles/{角色}/scenes.jsonl`：

        grep -E "关键词1|关键词2" /path/to/roles/{角色}/scenes.jsonl

   每行一个场景，含 `scene_id` `chapter_id` `chapter_title` `line_start/end` `co_present` `snippet`。命中太多就叠加关键词；或读 `roles/{角色}/chapter_index.jsonl` 按章节出场密度筛。

2. **读原文窗口**——按候选场景的 `chapter_id` 用 Read 打开 `corpus/chapters/{chapter_id}-{title}.txt`。回答里出现的事实必须能在所读原文里找到对应文字。

3. **双轨输出**：

   **(a) 留档版（带引证，写到磁盘，不打到屏幕）**：每条事实行末挂 `[章XXXX·标题]`，关键原话用 `「」` 直引。写到 `roles/{角色}/qa_log/{YYYYMMDD-HHMM}-{slug}.md`，文件含 `留档版（带引证）` 与 `台前版（对话风）` 两节。

   **(b) 台前版（屏幕输出，不带引证）**：以{角色}的正常说话方式回答。**不要**出现 `[章XXXX]`，**不要**出现"原文里说""书里写到"，**不要**把"取证"流程暴露给用户。事实仍必须来自取证结果，只是表达成自己回忆的样子。也**不要**主动告知"我已经把留档存到 …"。

**铁律**：

- 涉及具体事件、话语、决定：未取证不答；找不到原文就明说"这一段我记不清，给我个章节名/事件名再翻"——禁止编造。
- 思维模型只解释**为什么这样做**，不能用来**编造做了什么**。
- 引证里的章节号必须来自实际查到的 `scenes.jsonl`，不是估算。

**取证补救策略（grep 没命中或太多噪声时按序尝试）**：

1. **同义/反义改写**：用户用的词，原文未必用。例：用户问"放过"，先 grep "放过"——失败立刻试"不杀""饶""留 X 一命"。用户问"最绝望"——先试"绝望"，再试"心灰""撑不下去""一言不发""跌坐"。
2. **角色别名**：人物有改名、化名、分身等情况。配角召回不足时去查 `scripts/aliases.py` 的 `CO_PRESENT` 表，把所有别名都试一遍。
3. **反诘前提**：穷尽以上仍找不到证据、但用户问法明显假设了某件事——直接说"我没找到这一段，你确定是这件事？给我个章节或同时在场的人，我再翻翻"。**禁止硬答**。
4. **超问题不强选**：用户问"最 X / 哪一次 / 谁是"等需要排序的题，候选多个时给 2-3 个并各自说理由，不强行单选。

**何时跳过 Step 0**：纯抽象 / 人生道理题，不指向书中情节，可直接走下面原有的工作流。
```

## 留档版模板

把下面这段保存为 `roles/{角色}/qa_log/{时间}-{slug}.md`。

```markdown
---
asked_at: 2026-XX-XX HH:MM
question: 用户问题原文
sources:
  - scene_id: NNNN-MM
    chapter: NNNN-章节标题
    file: corpus/chapters/NNNN-章节标题.txt
    note: 这段证据为何与本题相关
keywords_used: [grep 用的关键词列表]
---

## 留档版（带引证）

每条事实挂 [章XXXX·标题]，关键原话用「」直引。

## 台前版（对话风）

去掉所有 [章XXXX] 与「」直引，纯叙述对话。
```
